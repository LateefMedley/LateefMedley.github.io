Project Name | League of Legends Gradient Boosting Classifier vs Multi-Layer Perceptron Model
Provide a quick summary of the game and how it works. Performed exploratory data analysis and created models using the first ten minutes of a match to find the most likely winner and uncover some other tips for where teams should invest their efforts early game. This includes an explanation of the multicollinearity problem faced, feature engineering/selection, using the t-SNE algorithm to in summary visualize the dimensional separability between games where the blue team won or lost, model training, practical uses, and recommendation for future work.
League of Legends

Project Name | How we can Learn from Tracking Fatal Encounters
Created a program that automatically retrieves data made public by fatal encounters via google sheet (so data retrieved is always current), reformat data into a time series depending on the geographical area of interest (Nationally, State, City, County, or Zipcode), presents the user of a quick analysis of previous fatal encounters via graphs, automatically uses fbProphet’s open-source time-series forecasting library to create a handful of model configurations that are tested using a commonly used machine learning technique known as train_test_split, the best model is then used to predict the number of fatal encounters 50 days from the current date, and return that information in a few forms. First, a graphic of all fatal encounters in the area of interest along with upper and lower limit predictions are shown. Second, a list of all the fatal encounters that happened in that area already are returned in an Excel spreadsheet format. The last two Excel documents are the actual time series of the area of interest and the excel spreadsheet of the predictions. One last graphic is shown inside the program, it displays the trend and seasonality of fatal encounters considering holidays, weekly, and annual seasonality.
Automated Fatal Encounter Modeler
